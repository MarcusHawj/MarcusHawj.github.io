Arena

MAY – JUL 2026
real-time engine

I wanted to understand graphics from the ground up, so I built a first-person shooter with no game engine — just C++17, OpenGL, and SDL2. Everything you see is rendered from a single instanced cube mesh; the variety comes entirely from the shaders.

  • Custom GLSL shaders drive all rendering from one instanced cube mesh — no external models.
  • Per-axis AABB collision so you slide along walls instead of sticking, plus slab-method raycasting for hitscan weapons and momentum-based air control.
  • Enemy AI built on a wander/chase state machine with boids-style separation and line-of-sight-gated ranged attacks.

DSA Visualizer

MAY – JUN 2026
algorithms / visualization

A desktop app for watching algorithms run. Eight sorting algorithms and BFS/DFS graph traversal animate step by step, drawn by a rendering pipeline I built from scratch in Java Swing.

  • 8 sorting algorithms (Bubble, Merge, Quick, Heap, Radix, and more) with live bar animations, step counters, and pause/resume controls.
  • Thread-safe execution using daemon threads and an AtomicBoolean state model that keeps the Swing EDT unblocked during live visualizations.
  • Modular 5-class architecture with custom-painted components, a token-based theme system, and 4 graph presets (random, grid, binary tree, and more).
